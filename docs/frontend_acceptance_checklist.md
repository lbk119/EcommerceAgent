# 前端手动验收清单

> 适用范围：EcomPilot 前端页面与 Gateway/Brain 闭环验收。建议在已启动 Brain、Gateway、UI 后执行。

## 1. 登录

操作步骤：
1. 打开前端地址，例如 `http://127.0.0.1:5173/login`。
2. 输入已注册账号和密码。
3. 点击“登录工作台”。

期望结果：
1. 登录成功后进入工作台或 onboarding 页面。
2. 登录失败时页面展示明确错误信息。
3. 401 或 token 失效后，本地会话被清理并跳转登录页。

## 2. Onboarding

操作步骤：
1. 使用新账号注册并进入初始化流程。
2. 填写店铺名称、主营类目、店铺类型和经营阶段。
3. 选择一个或多个平台。
4. 选择“使用示例数据体验”或保持上传模式。
5. 完成数字员工选择并点击“完成并进入工作台”。

期望结果：
1. 页面进入工作台，不长时间卡死。
2. 当前店铺显示为新建店铺。
3. 若选择示例数据，后续刷新后工作台出现商品、报告、活动和策略数据。

## 3. 导入示例数据

操作步骤：
1. 进入“数据导入”。
2. 选择“使用示例数据”。
3. 点击“执行导入”。

期望结果：
1. 页面提示示例经营数据已导入。
2. 工作台指标刷新，GMV、订单、商品数不再全为 0。
3. 导入历史出现一条 sample 记录。
4. 经营报告中心新增或保留导入后经营概览报告。

## 4. 工作台

操作步骤：
1. 进入“工作台”。
2. 查看指标卡、日报摘要、库存风险、活动复盘建议、策略进化审核。
3. 在无数据账号下再次查看工作台。

期望结果：
1. 有数据时展示 GMV、订单、转化率、库存风险等真实聚合值。
2. 无数据时指标为 0，并提示先导入数据或完成平台授权。
3. 页面可滚动，顶部和主体内容不遮挡。

## 5. 商品分析

操作步骤：
1. 进入“商品分析”。
2. 点击“生成商品优化方案”。
3. 观察按钮 loading、顶部任务提示和任务完成/后台执行提示。

期望结果：
1. 按钮调用真实 `/api/v1/products/analyze`。
2. 任务创建后页面显示“正在分析数据”。
3. 任务完成后工作台刷新；超时则提示任务仍在后台执行，不显示失败。
4. 商品表格显示商品名称、SKU、价格、库存、销量、转化率、分层和建议。

## 6. 库存风险

操作步骤：
1. 进入“库存风险”。
2. 点击“生成补货建议”。
3. 查看库存表格和任务提示。

期望结果：
1. 按钮调用真实 `/api/v1/inventory/replenishment-plan`。
2. 库存风险行展示 SKU、库存、风险等级、风险原因、建议动作。
3. 任务完成后可生成或更新库存类报告。

## 7. 活动复盘

操作步骤：
1. 进入“活动复盘”。
2. 有活动数据时点击“生成复盘报告”。
3. 无活动数据时点击同一按钮。

期望结果：
1. 有活动数据时调用真实 `/api/v1/campaigns/{id}/review`。
2. 无活动数据时页面提示“当前没有可复盘的活动数据”，不发起无效任务。
3. 活动表格展示评分、ROI、GMV、转化变化和 AI 复盘结论。

## 8. 报告详情

操作步骤：
1. 进入“经营报告”。
2. 点击任意报告行或“查看详情”。
3. 在无报告账号中进入报告页。

期望结果：
1. 报告详情调用真实 `/api/v1/reports/{id}`。
2. `contentMarkdown` 非空；草稿或生成中报告展示生成中说明。
3. 无报告时显示“暂无经营报告”，并引导生成管理层周报。

## 9. 策略审核

操作步骤：
1. 进入工作台的“策略进化审核”。
2. 对 pending 策略点击“接受”“暂缓”或“驳回”。
3. 刷新工作台。

期望结果：
1. 操作调用真实 `/api/v1/agents/strategies/{id}/approve|defer|reject`。
2. 成功后策略从 pending 列表消失。
3. 不存在的 strategy 返回 404，并在前端显示明确错误。

## 10. 退出登录

操作步骤：
1. 进入“个人中心”。
2. 点击“退出登录”。
3. 尝试返回工作台 URL。

期望结果：
1. 本地 session 被清理。
2. 页面跳转到登录页。
3. 未登录访问工作台会被路由守卫重定向到登录页。
